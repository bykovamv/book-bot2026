# 📚 Book Bot - Мониторинг и запуск

## 🚀 Быстрый старт

### 1. Проверка работоспособности
```bash
py health_check.py
```
Проверяет соединение с Telegram API и корректность токена.

**Если видите "Таймаут"** - нужен прокси. См. [PROXY_SETUP.md](PROXY_SETUP.md)

### 2. Настройка прокси (если нужно)
Если Telegram заблокирован в вашей сети:

1. Найдите рабочий прокси
2. Добавьте в `.env`:
```env
PROXY_URL=http://proxy.example.com:8080
```

### 3. Запуск бота с мониторингом
```bash
py monitor.py
```
- Автоматический перезапуск при падении
- Логирование в `logs/bot.log`
- Экспоненциальная задержка между перезапусками
- Остановка после 10 падений за 5 минут

### 4. Просмотр логов
```bash
py view_logs.py
```
Показывает последние 50 строк и следит за новыми записями.

## 📁 Структура файлов

```
book_bot/
├── bot.py              # Основной скрипт бота
├── monitor.py          # Мониторинг и автоперезапуск
├── health_check.py     # Проверка работоспособности
├── view_logs.py        # Просмотр логов в реальном времени
├── PROXY_SETUP.md      # Инструкция по настройке прокси
├── MONITORING.md       # Этот файл
├── .env                # Токен бота и прокси (не коммитить!)
├── .env.example        # Пример .env файла
├── requirements.txt    # Зависимости
└── logs/               # Папка с логами (создаётся автоматически)
    ├── monitor.log     # Лог монитора
    └── bot.log         # Лог бота
```

## 🔧 Команды

| Команда | Описание |
|---------|----------|
| `py health_check.py` | Проверка подключения к Telegram |
| `py monitor.py` | Запуск бота с автоперезапуском |
| `py view_logs.py` | Просмотр логов в реальном времени |
| `py bot.py` | Прямой запуск бота (без мониторинга) |

## ⚙️ Настройка мониторинга

В `monitor.py` можно изменить:
- `MAX_RESTARTS = 10` - макс. перезапусков за период
- `RESTART_WINDOW = 300` - окно в секундах (5 мин)
- Задержка между перезапусками растёт экспоненциально (2с, 4с, 8с... макс 60с)

## 🐛 Решение проблем

### Бот не запускается
1. Проверьте токен: `py health_check.py`
2. Посмотрите логи: `py view_logs.py`
3. Убедитесь, что `.env` файл существует

### Частые падения
- Проверьте логи в `logs/bot.log`
- Убедитесь, что токен валидный
- Проверьте интернет-соединение

### Остановка монитора
Нажмите `Ctrl+C` в терминале
